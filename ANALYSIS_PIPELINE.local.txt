ReturnSense — Analysis pipeline (reference; matches repo code as of last update)
================================================================================

================================================================================
SECTION A — PIPELINE AS IMPLEMENTED IN CODE
================================================================================

OVERVIEW
--------
1. Extension collects DM messages (scroll-capture or visible), assigns role + attribution_confidence per line, POSTs to /api/analyze.
2. API: auth → attribution sanity (422 if unreliable) → cache by conversation_hash → validators → tri or single-model AI → computeFinalScore (weights + network ceiling) → synthesis LLM → save buyer + signals → JSON.
3. GET /api/stats: risk_trend_7d — last 7 calendar days, per day: avg_trust (null if no analyses), high_risk_count (high+critical), analyses_count.

EXTENSION ATTRIBUTION (extension/content.js — harvestVisibleMessages)
-----------------------------------------------------------------------
• Primary: compare message bubble center X to the center of `[role=main]` (the DM column), not
  `window.innerWidth/2` (Instagram’s thread is a narrow column; viewport mid misclassifies everything).
• Bubble box: `rsPickMessageBubbleRect` — widest `div[dir=auto]` in the row (full-width row rects are ignored).
• LTR: center right of thread mid → seller (you); left → buyer. RTL: mirrored.
• Flex (justify-content end/start) is secondary: flex-start-only never implies buyer (column flex-start
  affected all rows). flex-end-only can still mark seller; both axes → deepest wins if geometry failed.
• Last resort: viewport-centered geometry. Confidence ~0.88 thread geometry, ~0.62 viewport, ~0.82 flex-only.

SERVER ATTRIBUTION (apps/web/lib/analysis/attribution.ts)
---------------------------------------------------------
• ATTRIBUTION_HIGH = 0.72, ATTRIBUTION_LOW = 0.55.
• Confirmed buyer transcript = buyer role + confidence ≥ 0.72; same for seller.
• Medium band 0.55–0.72 → uncertain background only; below 0.55 → unattributed metadata.
• checkAttributionSanity: if total ≥ 5 messages and (buyer_high === 0 OR seller_high === 0) → 422 ATTRIBUTION_ONE_SIDED.

TRI ENGINE (OPENROUTER_API_KEY set, RS_USE_TRI_ENGINE !== "false")
------------------------------------------------------------------
• apps/web/lib/ai/tri/run.ts — three parallel OpenRouter JSON calls with buildTriSharedContext:
  - Network block from formatNetworkProfileForPrompt(buildNetworkProfilePayload(network_ig, distinct_sellers)).
  - Confirmed buyer, confirmed seller, uncertain transcripts only (no mixed primary blob).
• Models: RS_TRI_MODEL_BEHAVIOR, RS_TRI_MODEL_COMMITMENT, RS_TRI_MODEL_FRAUD (env defaults in run.ts).
• aggregateTriSignals: commitment depth, fraud pattern strengths 0–3, combination risk, hard stops; conflict-resolution.ts CR-1..CR-3 (seller-safe).
• analyst_notes from models cleared before synthesis; aggregate sets empty notes.

SYNTHESIS (Improvement Eight)
-----------------------------
• apps/web/lib/ai/tri/synthesize.ts after computeFinalScore: OpenRouter JSON → analyst_notes + recommendation.
• Optional RS_TRI_MODEL_SYNTHESIS.

FALLBACK AI (Gemini or OpenRouter single-model)
-----------------------------------------------
• apps/web/lib/ai/gemini.ts, openrouter.ts, rs-analyst-v1.ts — same split transcripts + {NETWORK_BLOCK} in template.

SCORING (apps/web/lib/risk/score-engine.ts)
-------------------------------------------
• Dynamic weights: if seller historicalData.length ≥ 3 → higher history weight; Pakistan phone_country bumps phone slice; weak phone bumps address; strong phone + geocoded address slightly reduces address weight; normalize to sum 1.
• computeNetworkScoreCeiling(networkIgRow): structural cap when cross-seller fake outcomes pass thresholds.

NETWORK (apps/web/lib/network/network-layer.ts)
-----------------------------------------------
• getNetworkIgStats, getDistinctSellerCountForIg, buildNetworkProfilePayload, computeNetworkTrustLabel/Score.
• API returns network_profile; extension renders banner first; DB raw may include signal_conflicts_resolved.

API ROUTE (apps/web/app/api/analyze/route.ts)
---------------------------------------------
• checkAttributionSanity before AI; 422 with error, code, attribution counts.
• Merged ai_raw: attribution_summary (high/medium/unattributed), network_profile, conflict_resolutions, synthesis.
